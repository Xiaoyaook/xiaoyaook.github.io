回顾一下，使用git pages 和 Hexo创建blog的过程

配置SSH keys

（各大网站均有教程)
如果不存在就没有关系，如果存在的话，直接删除.ssh文件夹里面所有文件
其中
Enter passphrase (empty for no passphrase):<输入加密串>
Enter same passphrase again:<再次输入加密串>
这里在回车中会提示你输入一个密码，这个密码会在你提交项目时使用，如果为空的话提交项目时则不用输入。这个设置是防止别人往你的项目里提交内容。

添加SSH Key到GitHub

测试设置是否成功

Github上建仓库

注意project name：username.github.io 项目名称是固定的 必须为用户的username

安装Hexo

$ npm install -g hexo -g的意思是在全局环境下

部署hexo

建立一个名字叫「Hexo」的文件夹，然后在此文件夹中右键打开Git Bash

$ hexo init
Hexo随后会自动在目标文件夹建立网站所需要的所有文件
*但这样完成后不能执行hexo d 原因在Hexo 3.0版本后deploy git 被分开的，所以需要安装，安装命令如下:npm install hexo-deployer-git –save
现在已经搭建起本地的hexo博客了，执行
$ hexo g
$ hexo s
然后到浏览器输入localhost:4000 即可浏览hexo的初始界面

之后，将本地的博客同步到github

修改配置文件

到你刚刚创建的Repository下，找到https url 复制
编辑_config.yml文件
修改文件里面的deploy。其中的repository就改成你刚刚复制的地址。
Type为 git
Branch为master（注意与冒号间要空一格）
保存这个文件。

键入指令：

hexo generate
hexo deploy
Username是你的Github账号名称，而不是邮箱；Password就是你的Github的密码。

注意：每次修改本地文件后，需要键入hexo generate才能保存。每次使用命令时，都要在C:\Hexo目录下。
每次想要上传文件到Github时，就应该先键入hexo generate保存之后，再键入hexo deploy。

Hexo具有多种主题，我选用的是yilia

安装方法 $ git clone (https://github.com/litten/hexo-theme-yilia.git themes/yilia)
配置 修改hexo根目录下的 _config.yml ： theme: yilia
更新

cd themes/yilia
git pull

Hexo上传README文件

README.md放在source文件夹，然后修改全局config里面的skip-render: [README.md]

常用命令：

hexo new “postName” #新建文章
hexo new page “pageName” #新建页面
hexo generate #生成静态页面至public目录
hexo server #开启预览访问端口（默认端口4000，’ctrl + c’关闭server）
hexo deploy #将.deploy目录部署到GitHub

hexo现在支持更加简单的命令格式了，比如：

hexo g == hexo generate
hexo d == hexo deploy
hexo s == hexo server
hexo n == hexo new

自定义404页面仅对绑定顶级域名的项目才起作用，GitHub默认分配的二级域名是不起作用的

目前先将就用默认分配的域名，域名购买与DNS设置以后有机会再弄
